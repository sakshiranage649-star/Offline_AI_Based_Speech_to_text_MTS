=====================================================
TRANSLY - OFFLINE MULTILINGUAL TRANSLATOR
PROJECT REPORT & EXPLANATION FOR TEACHERS
=====================================================

1. WHAT IS TRANSLY?
-----------------------------------------------------
Transly is an advanced, 100% offline Artificial Intelligence (AI) application. It allows users to translate text, direct microphone speech, audio files, videos, and PDF documents between multiple regional and international languages. The biggest achievement of this project is that it runs entirely on local hardware—meaning it requires absolutely NO internet connection to process translations, ensuring maximum data security and privacy.

2. HOW DOES THE PROJECT WORK? (THE ARCHITECTURE)
-----------------------------------------------------
Our project is divided into three main layers:

A) Frontend (The Face): 
Built using HTML, CSS (Tailwind), and JavaScript. This is where the user uploads files, types text, or speaks into the microphone. It sends this data to the backend.

B) Backend (The Brain): 
Built using Python and the FastAPI framework. When a user uploads a video or audio file, the backend takes it, processes the media, runs the AI algorithms on it, and sends the final translation back to the screen.

C) Database (The Memory): 
We use a local MySQL Database to store translation history. This allows users to search up previous translations so they don't have to translate things twice.


3. THE ALGORITHMS WE USED (CORE AI TECHNOLOGY)
-----------------------------------------------------
We used Neural Machine Translation (NMT) and Transformer-based algorithms. Our AI backbone operates on two main models:

Model 1: OpenAI "Whisper" (For Speech-to-Text)
- What it does: If a user uploads an MP4 video or speaks into the mic, Whisper listens to it and creates a text transcript.
- How the Algorithm Works: It uses an "Encoder-Decoder Transformer Architecture". When video/audio is uploaded, our Python code (using MoviePy and Librosa libraries) extracts the audio waves and turns them into a mathematical picture called a "Spectrogram". The Whisper sequence-to-sequence algorithm scans this spectrogram, detects phonetic patterns, and outputs perfectly matched text.

Model 2: Meta "M2M100" 1.2B (For Text Translation)
- What it does: It takes the text provided by the user (or the text generated by Whisper from a video) and translates it perfectly into the target language.
- How the Algorithm Works: This is a "Many-to-Many Multilingual Neural Machine Translation" algorithm. Unlike older algorithms (like Google Translate used to be) which translated Hindi -> English -> French, our M2M100 algorithm skips English altogether. It uses Neural Networks to map the meaning of words into a high-dimensional mathematical space, directly translating from source to target without an intermediate language block.


4. HOW A VIDEO TRANSLATION WORKS (FLOWCHART IN WORDS)
-----------------------------------------------------
If a teacher asks: "How exactly do you translate a video?" Explain these 4 steps:
Step 1 (Upload): The user uploads an MP4 video file.
Step 2 (Extraction): Our Python backend uses "MoviePy" to split the audio track out of the video frames.
Step 3 (Transcription): The "Librosa" library formats the audio, and the "Whisper" AI algorithm listens to it and types out exactly what is being said in the video into raw text.
Step 4 (Translation): The "M2M100" AI algorithm takes that raw text, translates it into the desired language, saves it to the MySQL database, and sends it back to the computer screen.


5. WHY IS OUR PROJECT SPECIAL? (CONCLUSION)
-----------------------------------------------------
Most modern AI tools (like ChatGPT or Google Cloud) process data on remote corporate servers. This costs money and risks privacy. Transly is special because all the heavy Neural Network computation happens locally on the computer's CPU/GPU. It proves that advanced AI video and speech processing can be achieved securely in remote or offline environments.
